# FloralMatch

Веб-сайт цветочного магазина с конструктором букетов.

## Возможности

- **Каталог** — фильтрация товаров по типу цветка, цвету, поводу, размеру и цене с пагинацией
- **Карточка товара** — состав букета, выбор размера (малый/средний/большой) с автоматическим пересчётом цены, рекомендации по уходу
- **Корзина и оформление заказа** — гостевая корзина через идентификатор сессии, автоматическое объединение с пользовательской при входе
- **Визуальный конструктор** — ручная сборка букета на холсте: перетаскивание цветов, поворот, масштабирование, выбор упаковки и ленты, скриншот итоговой композиции для администратора
- **Подсказки флориста** — анализ цветовой гаммы, проверка совместимости цветов и карточки ухода для текущего букета
- **Автоподбор** — фильтрация готовых букетов из каталога по поводу, типу цветка и цвету
- **AI-помощник** — подбор букетов по текстовому описанию на естественном языке через языковую модель YandexGPT-lite
- **Личный кабинет** — история заказов, редактирование профиля, список избранного
- **Административная панель** — управление заказами, ассортиментом и пользователями

## Технологический стек

### Backend

- **Python 3.9** + **FastAPI** — REST API
- **PostgreSQL 16** — реляционная СУБД
- **SQLAlchemy** — ORM
- **Alembic** — миграции схемы базы данных
- **Pydantic** — валидация данных
- **python-jose** + **passlib (argon2)** — JWT-аутентификация и хэширование паролей
- **Uvicorn** — ASGI-сервер
- **pytest** — интеграционное тестирование
- **YandexGPT-lite** (Yandex Cloud Foundation Models API) — языковая модель для AI-помощника

### Frontend

- **React 19** + **Vite** — UI-библиотека и сборщик
- **React Router** — маршрутизация SPA
- **Axios** — HTTP-клиент
- **html2canvas** — снимки рабочей области конструктора
- **Framer Motion** — анимации интерфейса

## Запуск проекта

### Требования

- Python 3.9 или выше
- Node.js 18 или выше
- PostgreSQL 16
- Аккаунт Yandex Cloud с доступом к Foundation Models (для AI-помощника)

### Backend

1. Создать базу данных PostgreSQL:

   ```bash
   createdb floralmatch
   ```

2. Скопировать файл переменных окружения и заполнить значения:

   ```bash
   cp .env.example .env
   ```

   В файле `.env` указать:
   - `DATABASE_URL` — строка подключения к PostgreSQL
   - `SECRET_KEY` — секретный ключ для JWT
   - `YANDEX_API_KEY` — ключ API Yandex Cloud
   - `YANDEX_FOLDER_ID` — идентификатор каталога Yandex Cloud

3. Установить зависимости:

   ```bash
   pip install -r requirements.txt
   ```

4. Применить миграции:

   ```bash
   alembic upgrade head
   ```

5. Заполнить базу данных тестовыми данными:

   ```bash
   PYTHONPATH=backend python3 -m app.scripts.seed
   ```

6. Запустить сервер:

   ```bash
   python3 -m uvicorn main:app --reload
   ```

   Backend будет доступен на `http://localhost:8000`.
   Swagger UI с документацией API — на `http://localhost:8000/docs`.

### Frontend

1. Перейти в папку фронтенда:

   ```bash
   cd frontend
   ```

2. Установить зависимости:

   ```bash
   npm install
   ```

3. Запустить сервер разработки:

   ```bash
   npm run dev
   ```

   Frontend будет доступен на `http://localhost:5173`.

## Архитектура

Система построена на основе подхода SPA + REST API. Клиентская и серверная части полностью разделены и взаимодействуют через JSON. Бэкенд организован по принципу разделения ответственности: API-маршруты, валидация (Pydantic), бизнес-логика, ORM-модели. Глобальное состояние клиента управляется через три React-контекста: AuthContext, CartContext, FavoritesContext.

## Особенности реализации

- **Гостевая корзина** — идентификация неавторизованного пользователя через UUID-сессию в заголовке `X-Session-ID`. При входе в аккаунт гостевая корзина автоматически объединяется с пользовательской.
- **Фиксация цены** — в момент оформления заказа цена товара сохраняется в поле `price_at_purchase`, что гарантирует неизменность суммы заказа при последующем обновлении прайс-листа.
- **Вариативный состав букета** — поле `composition` в таблице товаров хранится в формате JSONB, что позволяет описывать произвольный состав без дублирования записей.
- **Размерная сетка букетов** — поле `sizes` (JSONB) содержит три варианта размера с разными ценами и количеством цветов.
- **Скриншот букета** — при оформлении заказа через визуальный конструктор библиотека `html2canvas` создаёт изображение рабочей области, которое прикрепляется к заказу для администратора.
- **AI-помощник** — текстовый запрос пользователя на русском языке передаётся языковой модели YandexGPT-lite, которая извлекает структурированные параметры фильтрации (типы цветов, цвета, повод). Извлечённые значения автоматически применяются как фильтры каталога.
